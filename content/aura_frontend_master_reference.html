Requirement 1Enterprise Data Grid
Server-side pagination
Ek baar mein sirf 50 items fetch karo. Browser mein 50,000 rows kabhi render mat karo — DOM crash hoga. API ko ?page=1&limit=50 bhejte rehna.
3 column filters
Category → Dropdown | Stock Level → Range Slider (e.g. < 20) | Price → Min–Max input. Teeno filters combine hokar ek API call bnate hain.
Debounced omnisearch (500ms)
Har keystroke pe API call nahi. 500ms ruko, phir query bhejo. useDebounce hook banao ya lodash use karo.
Sticky header + sortable columns
Header scroll pe fix rehna chahiye (position:sticky). Column header click pe sort toggle ho — asc → desc → none. Sort state URL params mein rakho.
Requirement 2Command Center — Analytics Dashboard
Widget 1 — Restock bar chart
Top 10 lowest-stock products. Bar chart (Recharts BarChart). Y-axis = product name, X-axis = stock count. Red/amber colors for urgency.
Widget 2 — Portfolio pie chart
Category-wise inventory valuation. Recharts PieChart ya Doughnut. Tooltip pe % aur ₹ value dikhao.
Widget 3 — KPI summary cards
3 cards: Total SKUs | Total Inventory Value ($) | Out of Stock Items. Big number + label format. API se ek hi endpoint se aane chahiye.
Requirement 3Export Module
Filtered CSV export
Jo data abhi table mein dikh raha hai — wahi export ho, full 50,000 nahi. Current filters + search + sort apply hoke data lao, phir Blob → URL.createObjectURL se download karo.
State & Data fetching
TanStack Query
Server state, caching, pagination
Automatic background refetch, stale-while-revalidate, aur keepPreviousData — page badlane pe data flash nahi hoga. Redux se zyada simple hai server data ke liye.
Zustand
Filter state, UI state
Filter values (category, stock, price, search, sort) globally share karne ke liye. Context API pe re-renders zyada hote. Zustand lightweight hai aur DevTools support hai.
Performance utils
use-debounce (npm)
Omnisearch 500ms delay
Custom hook likhne se safer. useDebouncedValue(search, 500) — simple, tested, SSR safe. Lodash ke bina bhi kaam karta hai.
TanStack Table
Data grid logic
Headless table library — sort, filter state sab built-in. 50,000 rows ka concern nahi kyunki server-side pagination hai, lekin column sort logic handmade nahi likhna padega.
Charts
Recharts
Bar + Pie charts
React ke liye native — Chart.js DOM directly access karta hai jo React lifecycle se clash karta hai. Recharts component-based hai, responsive by default, aur bundle size kam hai.
Recharts Tooltip
Custom chart tooltips
Built-in CustomTooltip prop se apna JSX inject karo. Stock value ₹ format mein dikhane ke liye Intl.NumberFormat saath use karo.
Export
Papa Parse
CSV generation
Papa.unparse(data) — auto comma escaping, special characters handle, header row auto. Manual CSV string concat karna risky hai (commas in names break it).
URL.createObjectURL
File download trigger
Browser native — koi library nahi chahiye. Blob → anchor click → revokeObjectURL. FileSaver.js overkill hai iske liye.
DX & Styling
Tailwind CSS
UI styling
Utility-first — sticky header ke liye sticky top-0, responsive grid ke liye grid-cols-3. CSS file maintainability zyada hoti hai enterprise projects mein.
Next.js App Router
Routing + SSR
Filter state URL search params mein rakho (useSearchParams) — browser back button, shareable links, aur bookmark support milta hai automatically.
CriticalAuthentication ka koi mention nahi
Document mein login system, user roles, ya session management nahi bataya gaya. Koi bhi URL se dashboard access kar sakta hai?
Solution
Next.js middleware mein route protection lagao. NextAuth.js ya Clerk use karo. Backend se "warehouse manager" aur "executive" roles clarify karo — kyunki Executive sirf dashboard dekhega, Manager export bhi karega. Yeh Backend team se ek meeting mein decide karna hoga.
CriticalAPI contract define nahi hai
Frontend ko pata nahi ki endpoint kya hai — /api/inventory?page=1&search=xyz ya kuch aur? Response shape kya hogi? Error codes kya honge?
Solution
Backend se OpenAPI / Swagger spec maango before coding starts. Nahi mili toh MSW (Mock Service Worker) se mock API banao — frontend aur backend parallel develop hote hain. apiTypes.ts file mein request/response types define karo aur dono sides share karo.
MediumFilter + sort + search — reset logic undefined
Agar user page 5 pe hai aur naya filter lagata hai — kya page 1 pe auto-reset hoga? Document silent hai.
Solution
Rule banao: koi bhi filter/search change hone pe page automatically 1 pe reset ho. Zustand store mein filter change pe setPage(1) call karo. Yeh expected UX hai — nahi kiya toh "no results" bug lagega.
MediumExport — kitna data download hoga?
"Currently filtered table data" — sirf 50 rows jo screen pe hain, ya pure filtered dataset ke saare pages? Document unclear hai.
Solution
Client se pucho. Most likely answer: saara filtered data (not just page 1). Iske liye export pe ek alag API call karo — GET /api/inventory/export?filters=... jo CSV directly stream kare. UI mein loading spinner lagao aur size warn karo ("~5000 rows export honge").
MediumReal-time dashboard — refresh strategy missing
Document kehta hai "real-time analytics" — lekin WebSocket/polling ka koi mention nahi. KPI cards stale ho jayenge.
Solution
TanStack Query mein refetchInterval: 30000 (30 seconds) se start karo — simple aur kafi hai. WebSocket sirf tabhi add karo jab client explicitly "live" data maange. Dashboard pe last updated timestamp dikhao.
LowEmpty states aur error states design nahi
Agar koi SKU search se match nahi karta, ya API fail ho jaati hai — UI kya dikhayega? Document mein nahi bataya.
Solution
3 states design karo — Loading (skeleton rows), Empty ("No items match your filters" + clear filters button), Error ("Something went wrong" + retry button). Yeh ek shared <DataState /> component mein wrap karo jo teeno conditions handle kare.
LowMobile responsiveness undefined
"Web-based dashboard" bola — kya warehouse manager phone se bhi use karega? Data grid 10 columns pe mobile mein toot jaata hai.
Solution
Client se confirm karo ki minimum screen size kya hai. Temporary fix: min-width: 1024px likh do aur mobile pe "Desktop mode use karein" message dikhao. Proper mobile grid ek alag sprint mein lo.
